Paradata Validation and Upload

User manual

  1. When you open the application, you will see two upload buttons for each of the tables and you will be able to navigate through your folders to select the correct set of files.
  2. After selecting the files, please click the “Validate” button.

If there are mistakes in the files, 1 to 3 error logs will be generated. You can find them on your desktop:

Please fix all mistakes and mismatches and restart the process.

Table requirements:

  • File name: "Participants_XX.xlsx"/”CallHistory_XX.xlsx” - where XX is the country code.
  • No blank rows in the tables
  • Column names – case sensitive:
  • Sheet names: Participants/CallHistory
  • Call history ID needs to be present in the Participant table
  • Fresh status in PT - the ID should not be in the Call history
  • Call outcomes/Queue correlation in the Participant table
  • Try count in Participant Table needs to match the number of records in Call History
  • Call outcome for each ID in the Participant table needs to match the last outcome in the Call history
  1. If all of the requirements are met, the validation will be successful, and you will be able to enter your credentials for the secure FTP server.

  1. Once you have entered the correct username and password, you will be able to submit the files.

When you get the confirmation on your screen, the tables are uploaded and the process is successfully completed. You can now close the application or upload a new set of tables using the “Restart” option from the help menu.
